## ✨ 项目简介

班级宠物园是一个面向中小学教师的班级管理工具，将积分评价与电子宠物养成玩法相结合。学生通过良好表现获得积分，让宠物成长升级，最终获得毕业徽章。

### 核心特点

- 🎓 **多班级管理** - 支持创建和管理多个班级
- 🐾 **宠物养成** - 24种宠物，8级成长体系
- ⭐ **积分评价** - 83条默认规则，支持自定义
- � **积分商城** - 用可用积分兑换礼品，支持商品管理和兑换记录
- � **排行榜** - 实时积分排名
- 💾 **数据安全** - SQLite数据库，支持完整数据的备份和恢复
- 📱 **响应式设计** - 适配多端设备

## 🚀 快速开始

### 在线演示
https://8.147.56.12/pet-garden/

### 本地开发

```bash
# 安装依赖
npm install

# 启动前端和后端
npm start

# 或分别启动
npm run dev      # 前端 :3001
npm run server   # 后端 :3002
```

### 生产部署

项目使用 Nginx 托管静态文件，后端 API 独立运行。

**部署命令：**
```bash
# 构建并部署到 Nginx 静态目录
npm run deploy

# 查看用户和班级统计
npm run stats
```

**服务管理：**
```bash
# 查看后端服务状态
systemctl status pet-garden

# 重启后端服务
sudo systemctl restart pet-garden

# 查看日志
journalctl -u pet-garden -f
```

**Nginx 配置要点：**
- 静态文件：`/var/www/pet-garden/`
- API 代理：`/pet-garden/api/` → `http://127.0.0.1:3002`
- 已启用 gzip 压缩
- 静态资源缓存 1 年

### 宠物系统（24种）

| 类型 | 宠物 |

### 成长体系

| 等级 | 积分 | 称号 |
|------|------|------|
| Lv.1 | 0 | 初生 |
| Lv.2 | 40 | 成长 |
| Lv.3 | 100 | 优秀 |

| 分类 | 加分 | 扣分 |

### 积分系统

- **累计积分** (`total_points`)：用于宠物升级、排行展示
- **可用积分** (`usable_points`)：用于商城礼品兑换
- 加分时同时增加两种积分，扣分时只扣累计积分
- 撤回加分时同时减去可用积分（保证可用积分不为负）

### 积分商城

- 商品管理：增删改查、上架/下架、库存管理
- 商品兑换：选择学生、检查积分、扣减库存
- 兑换记录：查看所有兑换历史
- 支持无限库存（stock = -1）

## 🔌 API 接口

```
# 认证
POST   /api/auth/register        # 注册
POST   /api/auth/login           # 登录
GET    /api/auth/me              # 当前用户

# 班级
GET    /api/classes              # 班级列表
POST   /api/classes              # 创建班级
PUT    /api/classes/:id          # 更新班级
DELETE /api/classes/:id          # 删除班级
GET    /api/classes/:id/students # 学生列表
GET    /api/settings/ranking/:id # 排行榜

# 学生
POST   /api/students             # 添加学生
PUT    /api/students/:id         # 更新学生
DELETE /api/students/:id         # 删除学生
POST   /api/students/import      # 批量导入
PUT    /api/students/:id/pet     # 更换宠物

# 评价
GET    /api/rules                # 规则列表
POST   /api/rules                # 添加规则
DELETE /api/rules/:id            # 删除规则
POST   /api/evaluations          # 添加评价
GET    /api/evaluations          # 评价记录
DELETE /api/evaluations/:id      # 删除记录

# 商城
GET    /api/shop/products        # 商品列表
POST   /api/shop/products        # 添加商品
PUT    /api/shop/products/:id    # 更新商品
DELETE /api/shop/products/:id    # 删除商品
GET    /api/shop/redemptions     # 兑换记录
POST   /api/shop/redeem          # 兑换商品

# 数据
GET    /api/backup               # 导出备份（包含所有数据）
POST   /api/backup               # 导入恢复
GET    /api/health               # 健康检查
```

## 🆕 2026年4月更新 - 学生账号化与只读学生端

这次更新把“学生”从普通数据对象提升为可登录的用户类型，并明确区分教师端与学生端权限边界。

### 账号与权限重构

| 功能 | 说明 |
|------|------|
| **学生账号模型** | 老师添加学生时必须填写姓名、学号、密码；学号同时作为唯一 ID 和登录账号 |
| **单班级归属** | 每个学生账号只能属于一个班级，转班时会同步账号归属 |
| **只读学生端** | 学生登录后只能查看本班首页、排行、商城、图鉴，不能新增、修改、删除任何数据 |
| **教师写权限收口** | 评价、规则、标签、商品、备份、复活任务等写接口统一限制为教师可用 |
| **旧数据兼容** | 历史学生数据在老师重置密码时会自动补建对应的学生账号 |

### 教师端体验补充

| 功能 | 说明 |
|------|------|
| **学生账号管理页升级** | 支持新增、批量导入、编辑姓名、重置密码、删除学生账号 |
| **学生标签管理恢复** | 教师可在学生账号管理页直接为单个学生添加或移除标签 |
| **导航按角色裁剪** | 学生端隐藏学生管理、记录、设置、帖子等教师专属入口 |

### 数据与部署影响

- 数据库新增学生账号相关字段：`user_type`、`student_id`、`class_id`、`user_id`
- 学生删除、班级删除、备份恢复时会同步清理对应学生账号，避免孤立用户
- 后端测试与前端测试已同步更新，保证账号化后的流程可回归验证
